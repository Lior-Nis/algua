The architecture exists to keep that honest: pure, well-bounded modules, one definition of a signal, and a wall the agent cannot climb.

Two load-bearing invariants

Almost every structural decision in the codebase serves one of these two ideas. The rest of this map shows where they live.

① One signal def + one execution contract

A strategy is a pure function — features up to bar t → target weights. The backtester and the live loop call the same function. Parity of semantics, not PnL, is pinned by an explicit ExecutionContract: decide on closed bar t, fill no earlier than t+1. That single rule is the firewall against look-ahead bias.

② State is a record · the live gate is a wall

Lifecycle stage lives in a SQL registry the agent can write — so a bare stage='live' flag is forgeable by the very actor it should stop. paper → live therefore demands a human actor and an Ed25519 signature over a single-use challenge bound to the exact code + config + dependency identity, verified against a CODEOWNERS-protected trust anchor. The agent can request a challenge but never holds a key that signs it.

01 · Module architecture

The stack, bottom-up. Purity at the base.

Dependencies point inward and down. The two bottom layers import no other Algua code — that's what lets an agent (and you) reason about them in isolation. Click a module for its real role & files; trace its dependencies; or flash the purity boundary that lint-imports enforces.

Click any chip on the left. The base layers — contracts, calendar, features — are pure: stdlib + pandas only, zero cross-module imports. Everything above composes them.
The import rule, verbatim from the spec: strategies/ and features/ stay pure; backtest/ and live/ consume the same strategy contract but never each other's internals; agent-facing code never imports live-execution internals directly.
02 · Lifecycle & the live gate

Six stages. One wall.

The state machine from contracts/lifecycle.py, exactly. An agent advances a strategy up through paper on its own. The paper → live edge is the wall: a two-step signed ceremony — issue a single-use challenge, sign it with an enrolled key, verify against a code-owner-controlled trust anchor. Try it as an agent, or without a signature, and watch the real check refuse.

03 · Research → execution data flow

One function. Two runners. One contract.

The authored signal is defined once. The backtester replays it over history; the live/paper loop calls it on the latest closed bar. Both obey the same ExecutionContract spine — decide on t, fill at t+1.
