Two gates guard the path

① Promotion gate

backtested → shortlisted · agent-driven

A good-looking backtest isn't enough. research promote runs a walk-forward and judges an untouched out-of-sample holdout — promoting only if it clears statistical thresholds.

  • holdout Sharpe ≥ threshold & positive return
  • stability: enough positive windows, worst-window floor
  • promote only on pass; else the strategy stays backtested

② Live gate — the wall

paper → live · human-only

The registry stage is a record an agent can write, so a bare stage='live' flag is forgeable by the very actor it should stop. The transition therefore demands more than a flag:

  • actor must be human (agents are refused)
  • a recorded approval bound to the exact code + config hash, recomputed from source at transition time
  • the runner trusts the approval, never the bare stage flag
