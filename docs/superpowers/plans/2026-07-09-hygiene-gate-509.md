# Plan — Structural CI Hygiene Gate (#509)

Design: `docs/superpowers/specs/2026-07-09-hygiene-gate-509-design.md`.

This PR ships a deliberately narrow gate. The tighter boundary (top-level-dir enforcement, per-
segment junk matching, name agreement, report stamp, `algua strategy verify` + `doctor` probe) is
deferred to **#511** — see the design's §7.

Each task lists its FAST per-task check (lint/type the whole tree + run only this task's tests).
Full gate (`pytest -q && ruff check . && mypy algua && lint-imports`) runs at integration.

## Task 3 — The canonical structural test `tests/test_repo_hygiene.py` (static, DB-free)
**The core deliverable.** No import of any `algua.*` module that opens a connection; uses
`subprocess`+`git ls-files`, `pathlib`, `ast`, `fnmatch` only. Source of truth is the git index —
all three tests share `_tracked_files()`.
- Module constants: `ROOT_WHITELIST` (repo-root files, §5), `JUNK_PATTERNS` (basename patterns),
  `INFRA_TOP_LEVEL` (allowed strategies-top-level modules).
- `test_repo_root_is_whitelisted` — a tracked path with no `/` fails unless its basename is in
  `ROOT_WHITELIST`.
- `test_no_tracked_junk` — a tracked path whose basename matches any `JUNK_PATTERNS` entry fails.
- `test_strategies_placement_and_provenance` — over tracked `algua/strategies/**.py`: a non-infra
  module at the strategies top level is unplaced (fails); every module inside a family subdir must
  carry a module-level `GENERATED_BY` (AST `_has_generated_by`, presence-only).
- **Check:** `uv run ruff check . && uv run mypy algua && uv run lint-imports && uv run pytest -q tests/test_repo_hygiene.py`
